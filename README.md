# אתר מלווה — סדנת מצגות וחומרים ויזואליים · מנהלי בת ים

אתר סטטי (ללא שלב בנייה) שמלווה את הסדנה: מדריך צעד-אחר-צעד ופרומפטים מוכנים להעתקה,
לארבעה מסלולים — מסמך המתווה, ChatGPT Image2, NotebookLM, ו-Gemini Canvas.

## איך עורכים תוכן

כל התוכן יושב בקובץ אחד: **`content.js`**.
משנים שם טקסט/פרומפט/רעיון — והאתר מתעדכן. אין צורך לגעת ב-HTML.

מבנה כל מסלול ב-`content.js`:
- `eyebrow`, `title`, `subtitle`, `meta[]` — כותרת המסלול.
- `intro.paragraphs[]`, `intro.facts[]` — מקטע "מה זה בכלל?".
- `steps[]` — צעד-אחר-צעד. כל צעד: `title`, `text`, ואופציונלי `prompt` (מקבל כפתור העתקה) ו-`tip`.
- `ideas[]` — רעיונות. כל רעיון: `title`, `desc`, `useCase` (לסינון), `prompt`. אופציונלי `promptLabel`.
- `sources[]` — קישורי העמקה.
- `moeNote: true` — מציג את הערת הכניסה דרך משרד החינוך (רלוונטי ל-Canvas).

## הרצה מקומית

פותחים את `index.html` בדפדפן.
שימו לב: כפתורי ההעתקה (clipboard) עובדים על אתר מאובטח (https) — כלומר על הקישור החי,
לא תמיד על פתיחה מקומית כקובץ. לכן בודקים העתקה על כתובת ה-GitHub Pages.

## פריסה (GitHub Pages)

1. `git init` ודחיפה ל-GitHub (דרך הסקיל `/github`).
2. בהגדרות הריפו: Pages → Branch: `main` → `/ (root)`.
3. הקישור: `https://tomhagiladi.github.io/<repo-name>/`

## קבצים

- `index.html` — מעטפת: כותרת, ניווט, דף הבית עם 4 כרטיסים, תחתית עם הצהרת נגישות.
- `script.js` — ראוטר (hash), רינדור מסלולים, כפתורי העתקה, מסנן "סוג השימוש".
- `content.js` — כל התוכן.
- `styles.css` — מערכת עיצוב רשמית-נקייה (כחול-פטרול, Frank Ruhl Libre + Heebo).
- `assets/` — לוגו ואייקון.

## נגישות

ניגודיות תקנית, `aria-current`, כיבוד `prefers-reduced-motion`, והצהרת נגישות בתחתית.
